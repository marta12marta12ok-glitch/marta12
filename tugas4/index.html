1. Class Hewan


// Nama  : Marta Saputra
// NIM   : 251410020
// Kelas : SI2A

abstract class Hewan {
  String nama = "Hewan";
  void bersuara();
}

class Kucing extends Hewan {
  Kucing() { nama = "Kucing"; }
  void bersuara() => print('$nama: Meong');
}

class Anjing extends Hewan {
  Anjing() { nama = "Anjing"; }
  void bersuara() => print('$nama: Guk guk');
}

void main() {
  List data = [Kucing(), Anjing()];
  for (var h in data) {
    h.bersuara();
  }
}

6. Class Pesawat (Interface)

Konsep interface dalam Dart digunakan untuk mendefinisikan kontrak yang harus diimplementasikan oleh class lain.


// Nama  : Marta
// NIM   : 251410020
// Kelas : SI. 2A

abstract class Kendaraan {
  void jalan();
}

abstract class Terbang {
  void terbang();
}

class Pesawat extends Kendaraan implements Terbang {
  @override
  void jalan() {
    print('Pesawat berjalan di landasan');
  }

  @override
  void terbang() {
    print('Pesawat terbang di udara');
  }
}

void main() {
  Pesawat p = Pesawat();
  p.jalan();
  p.terbang();
}

7. Static Method

Static digunakan untuk variabel dan method yang dapat diakses tanpa membuat objek.


// Nama  : Marta
// NIM   : 251410020
// Kelas : SI. 2A

class MathUtils {
  static double pi = 3.14;

  static int tambah(int a, int b) {
    return a + b;
  }
}

void main() {
  print(MathUtils.pi);
  print(MathUtils.tambah(5, 10));
}

8. Metode Pembayaran (Interface)

Interface digunakan untuk membuat berbagai metode pembayaran dengan implementasi berbeda.


// Nama  : Marta
// NIM   : 251410020
// Kelas : SI. 2A

abstract class MetodePembayaran {
  void bayar(double jumlah);
}

class KartuKredit implements MetodePembayaran {
  @override
  void bayar(double jumlah) {
    print('Bayar Rp$jumlah pakai Kartu Kredit');
  }
}

class EWallet implements MetodePembayaran {
  @override
  void bayar(double jumlah) {
    print('Bayar Rp$jumlah pakai E-Wallet');
  }
}

void main() {
  KartuKredit k = KartuKredit();
  EWallet e = EWallet();

  k.bayar(50000);
  e.bayar(25000);
}

9. Bentuk Geometri

Menggunakan abstraksi untuk menghitung luas dan keliling berbagai bentuk.


// Nama  : Marta
// NIM   : 251410020
// Kelas : SI. 2A

abstract class Bentuk {
  double luas();
  double keliling();
}

class Persegi extends Bentuk {
  double sisi;
  Persegi(this.sisi);

  @override
  double luas() => sisi * sisi;

  @override
  double keliling() => 4 * sisi;
}

void main() {
  Persegi p = Persegi(4);
  print(p.luas());
  print(p.keliling());
}

15. Kesimpulan

Dari latihan-latihan di atas, dapat disimpulkan bahwa:

  • Abstraksi menyembunyikan detail implementasi dan hanya menampilkan fungsi penting kepada pengguna.
  • Polimorfisme memungkinkan kita menggunakan satu nama method untuk melakukan berbagai tindakan yang berbeda tergantung objeknya.
  • Inheritance (Pewarisan) memungkinkan pembuatan class baru berdasarkan class yang sudah ada, sehingga kode lebih efisien dan terstruktur.
  • Interface berfungsi sebagai kontrak yang memaksa class turunan untuk mengimplementasikan semua method yang didefinisikan di dalamnya.
  • Dengan menggabungkan konsep-konsep OOP (Object-Oriented Programming) ini, program menjadi lebih mudah dikembangkan, dipelihara, dan dimodifikasi di kemudian hari.
