01. Pengertian OOP

Object Oriented Programming (OOP) Object Oriented Programming (OOP) adalah paradigma pemrograman yang berorientasi pada objek, dimana program dibuat menggunakan class dan objek. Dalam OOP terdapat beberapa konsep utama seperti Encapsulation, Inheritance, Polymorphism, dan Abstraction yang membantu membuat program lebih terstruktur, modular, dan mudah dikembangkan.

02. Konsep Dasar

  • Class: Blueprint atau cetakan untuk membuat objek.
  • Object: Instance nyata yang tercipta dari sebuah class.
  • Attribute: Representasi data melalui variabel di dalam class.
  • Method: Logika fungsional atau fungsi di dalam class.

03. Class Sederhana (Truk)

Pada contoh program di bawah ini terdapat class Truk yang memiliki dua atribut yaitu merk dan warna yang digunakan untuk menyimpan spesifikasi kendaraan angkut. Selain itu terdapat dua method yaitu jalan() dan parkir() yang berfungsi sebagai aksi dari objek truk tersebut.

DART SOURCE CODE
// Nama   : Marta saputra
// Nim    : 251410020
// Kelas  : SI2A

class Truk {
  String? merk;
  String? warna;
  
  void jalan() {
    print('Truk sedang berjalan...');
  }
  
  void parkir() {
    print('Truk sudah diparkir.');
  }
}

void main() {
  Truk trukMarta = Truk();
  
  trukMarta.merk = "Scania";
  trukMarta.warna = "Perak";
  trukMarta.jalan();
  trukMarta.parkir();
}

04. Object Instantiation

Object Instantiation adalah proses pembuatan objek nyata dari sebuah class. Pada contoh berikut dibuat sebuah objek bernama bisUmum dari class Bis. Kemudian atribut rute dan jenis diatur nilainya untuk merepresentasikan sebuah armada transportasi umum.

DART SOURCE CODE
// Nama   : Marta saputra
// Nim    : 251410020
// Kelas  : SI2A

class Bis {
  String? rute; 
  String? jenis; 
}

void main() {
  Bis bisUmum = Bis();
  
  bisUmum.rute = "Blok M - Kota"; 
  bisUmum.jenis = "Tingkat";

  print("Rute Perjalanan: ${bisUmum.rute}");

07. Encapsulation

Encapsulation diimplementasikan menggunakan class Pegawai untuk menjaga privasi data gaji (_gaji). Variabel yang bersifat private ditandai dengan underscore agar tidak dapat diakses secara bebas secara eksternal, melainkan melalui getter method yang aman.

DART SOURCE CODE
// Nama   : Marta saputra
// Nim    : 251410020
// Kelas  : SI2A

class Pegawai {
  String nama;
  double _gaji; 

  Pegawai(this.nama, this._gaji);

  void updateGaji(double nilai) => _gaji = nilai;
  double get cekGaji => _gaji;
}

void main() {
  Pegawai staff = Pegawai("Marta Saputra", 5000000);
  print('Staff: ${staff.nama}'); 
  print('Gaji Saat Ini: ${staff.cekGaji}'); 
}

08. This Keyword

Keyword "this" digunakan untuk memperjelas identitas atribut milik class agar tidak terjadi kerancuan saat nama parameter sama dengan nama field. Pada class Modul berikut, this.judul memastikan bahwa nilai yang diproses adalah milik objek tersebut.

DART SOURCE CODE
// Nama   : Marta saputra
// Nim    : 251410020
// Kelas  : SI2A

class Modul {
  String judul; 
  Modul(this.judul);
}

void main() {

}

09. Product Class (Tiket)

Class Tiket menggunakan named parameters demi kejelasan data saat pembuatan objek. Penggunaan keyword 'required' memastikan atribut penting seperti acara dan harga selalu terisi, sementara kursi memiliki nilai default untuk kemudahan user.

DART SOURCE CODE
// Nama   : Marta saputra
// Nim    : 251410020
// Kelas  : SI2A

class Tiket {
  final String kode, acara; 
  final double harga; 
  final String kursi;   

  Tiket({
    required this.kode,
    required this.acara,
    required this.harga,
    this.kursi = 'Umum (Festival)',
  });
}

void main() {
  Tiket t1 = Tiket(
    kode: "T-001",
    acara: "Konferensi Teknologi",
    harga: 15000000,
    kursi: "VVIP Baris 1",
  );

  Tiket t2 = Tiket(
    kode: "T-002",
    acara: "Workshop Programmer",
    harga: 500000,
  );

  print('Acara 1: ${t1.acara} (Kursi: ${t1.kursi})'); 
  print('Acara 2: ${t2.acara}'); 
}
